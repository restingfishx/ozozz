# PRD 模板

## 标准 PRD (7 部分)

适用于：简单功能 (2-4 周)

```markdown
# [功能名称]

## 1. 问题与背景
- 解决的问题
- 用户痛点
- 当前 workaround

## 2. 目标与指标
- 成功指标（北极星指标）
- 预期效果

## 3. 用户故事
- 作为 [用户]，我希望 [功能] 以便于 [价值]

## 4. 功能列表
| 功能 | 优先级 | 验收标准 |
|------|--------|----------|
|      | P0/P1  |          |

## 5. 边界条件
- 包含什么
- **不包含什么**（明确 out of scope）

## 6. 技术考虑
- 依赖
- 风险

## 7. 发布计划
- 灰度策略
- 回滚方案
```

---

## 模板选择指南

| 格式 | 适用场景 | 建议时长 |
|------|----------|----------|
| **标准 PRD** | 简单功能 | 2-4 周 |
| **详细 PRD** | 复杂功能 | 6-8 周 |
| **特性简报** | 探索阶段 | 1 周 |
| **Agile Epic** | 迭代交付 | 多个 sprint |
